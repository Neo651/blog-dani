# Criterios para añadir nuevas funcionalidades

Antes de comenzar un desarrollo preguntar:

¿Ayuda a pensar mejor?

¿Mejora la lectura?

¿Hace regresar al lector?

¿Fortalece la marca?

¿Aporta valor real?

Si la respuesta es no, la funcionalidad debe esperar.

# Regla final

Nunca desarrollar funcionalidades únicamente porque están de moda.

Toda nueva característica deberá responder claramente:

¿Por qué mejora la vida del lector?

Si no existe una respuesta sólida, esa funcionalidad no debe desarrollarse.

# Oportunidades detectadas

No son funcionalidades.

Son problemas observados.

Ejemplo

2026-07-11

Muchos lectores abandonan después de terminar una reflexión.

Hipótesis:

No saben qué leer después.

Posibles soluciones:

- Reflexiones relacionadas.
- Colecciones.
- Continuar leyendo.
