# UX_UI_GUIDELINES.md

# DanielMontesinos.com

Versión: 1.0
Estado: Documento vivo

---

# 1. Filosofía

No diseñamos pantallas.

Diseñamos experiencias.

Cada elemento debe ayudar al lector a pensar mejor.

Todo aquello que distraiga debe eliminarse.

La mejor interfaz es aquella que desaparece para dejar que el contenido sea el protagonista.

---

# 2. Objetivo

El usuario no entra para navegar.

Entra para reflexionar.

La navegación existe únicamente para facilitar esa reflexión.

Nunca debe convertirse en protagonista.

---

# 3. Principios de UX

Toda decisión debe responder estas preguntas:

• ¿Reduce la fricción?

• ¿Reduce la carga cognitiva?

• ¿Hace más agradable la lectura?

• ¿Hace que el usuario quiera continuar?

Si una decisión no mejora alguno de estos aspectos, probablemente no deba implementarse.

---

# 4. Principio del silencio visual

El espacio en blanco no es espacio perdido.

Es descanso para la mente.

Siempre priorizar:

más aire

menos ruido

más claridad

menos elementos

---

# 5. Jerarquía visual

El usuario siempre debe saber qué hacer.

Debe existir un único protagonista por pantalla.

No competir entre botones.

No competir entre colores.

No competir entre títulos.

---

# 6. Home

La Home debe sentirse como la portada de un libro.

No como una lista infinita de artículos.

Debe responder en menos de cinco segundos:

Quién es Daniel Montesinos.

Qué encontrará aquí.

Por qué vale la pena quedarse.

---

# 7. Hero

El Hero debe transmitir calma.

Mucho espacio.

Texto corto.

Mensaje poderoso.

Nunca más de un botón principal.

El CTA debe invitar.

Nunca presionar.

---

# 8. Reflexión destacada

La reflexión destacada debe ocupar el centro emocional de la página.

Debe sentirse importante.

No como una tarjeta más.

El contenido debe respirar.

La lectura debe ser cómoda incluso durante varios minutos.

---

# 9. Lectura

La lectura siempre tiene prioridad.

Reglas:

• ancho cómodo

• excelente contraste

• tipografía grande

• interlineado amplio

• párrafos cortos

• imágenes limpias

Nunca sacrificar legibilidad por diseño.

---

# 10. Navegación

La navegación debe ser casi invisible.

Debe existir un menú simple.

Ejemplo:

Inicio

Reflexiones

Sobre mí

Contacto

Si el usuario está en la Home:

los enlaces harán scroll suave.

Si está en otra página:

volverán a la sección correspondiente.

---

# 11. Scroll

El scroll debe sentirse natural.

No obligar al usuario a hacer clic constantemente.

Siempre que sea razonable:

preferir scroll antes que abrir nuevas páginas.

---

# 12. Reflexiones anteriores

Después de la reflexión destacada deben mostrarse varias reflexiones recientes.

Cada tarjeta mostrará:

imagen

fecha

tiempo de lectura

título

frase

likes

comentarios

botón Leer

Después aparecerá:

Ver todas las reflexiones

---

# 13. Biblioteca

La biblioteca completa tendrá:

Buscador

Ordenar por:

Más recientes

Más populares

Más comentadas

Preparada para filtros futuros.

---

# 14. Página individual

Cada reflexión debe sentirse como un capítulo.

Al finalizar mostrar:

❤️ Likes

💬 Comentarios

Reflexiones recomendadas

Nunca terminar en una pantalla vacía.

---

# 15. Comentarios

No usar títulos fríos.

Preferir preguntas abiertas.

Ejemplo:

¿Qué reflexión te dejó este texto?

El objetivo es invitar a pensar.

No únicamente comentar.

---

# 16. Likes

El like representa conexión.

No popularidad.

El texto debe transmitir humanidad.

Ejemplo:

❤️ 128 personas conectaron con esta reflexión.

---

# 17. Sobre mí

No debe parecer un currículum.

Debe parecer una conversación.

Debe responder:

¿Por qué escribo?

¿Qué me mueve?

¿Por qué comparto estas ideas?

---

# 18. Redes sociales

Las redes no son publicidad.

Son continuidad.

Mostrar:

LinkedIn

Instagram

GitHub

Correo

Preparado para:

YouTube

Podcast

Newsletter

---

# 19. Formularios

Todo formulario debe:

estar centrado

tener ancho máximo

agrupar campos relacionados

mostrar validaciones claras

mostrar mensajes de éxito

mostrar mensajes de error útiles

---

# 20. Panel de administración

Inspiración:

Ghost

Notion

Medium

Nunca debe sentirse como un sistema antiguo.

Debe transmitir simplicidad.

---

# 21. Dashboard

Debe responder rápidamente:

¿Cuántas reflexiones tengo?

¿Cuántas publicadas?

¿Cuántos borradores?

¿Cuántos comentarios?

¿Cuántos likes?

¿Cuál fue la última publicación?

No saturar con métricas innecesarias.

---

# 22. Botones

Debe existir un botón primario.

Los demás serán secundarios.

Evitar más de un botón destacado por sección.

---

# 23. Iconografía

Usar pocos iconos.

Solo cuando mejoren la comprensión.

Nunca como decoración.

---

# 24. Animaciones

Las animaciones deben sentirse naturales.

Nunca llamar la atención.

Duración corta.

Movimiento suave.

---

# 25. Responsive

La experiencia móvil tiene la misma importancia que escritorio.

Nunca esconder funcionalidades importantes únicamente por ser móvil.

---

# 26. Accesibilidad

Contraste correcto.

Texto legible.

Botones suficientemente grandes.

Navegable mediante teclado.

Imágenes con texto alternativo.

---

# 27. Rendimiento

La velocidad forma parte de la experiencia.

No añadir librerías pesadas si no aportan valor.

Optimizar imágenes.

Evitar JavaScript innecesario.

---

# 28. Principios de diseño

Menos es más.

La claridad supera a la creatividad.

La simplicidad supera a la complejidad.

La lectura supera a la decoración.

---

# 29. Regla de oro

Antes de aceptar cualquier cambio de interfaz preguntar:

¿Hace que la experiencia sea más tranquila?

¿Hace que leer sea más agradable?

¿Reduce clics?

¿Reduce ruido?

¿Hace que el usuario quiera volver?

Si alguna respuesta es negativa, replantear el diseño.

---

# 30. Definición de una experiencia excelente

Una experiencia excelente es aquella donde el usuario deja de pensar en la interfaz.

Solo piensa en las ideas que acaba de leer.

Cuando la tecnología desaparece y la reflexión permanece, el diseño ha cumplido su objetivo.

# Principio de serenidad

Cada pantalla debe transmitir la sensación de que el usuario tiene tiempo.

Nunca debe sentirse apresurado.

Nunca debe sentirse presionado.

Nunca debe sentir que la página intenta venderle algo.

El sitio debe convertirse en un refugio dentro del ruido digital.

Si una decisión aumenta el estrés del usuario, debe rechazarse aunque sea técnicamente correcta.
