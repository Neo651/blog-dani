# PROJECT_MANIFEST.md

# DanielMontesinos.com

Versión: 1.0
Estado: Documento vivo
Última actualización: 2026-07-11

---

# 1. Propósito

DanielMontesinos.com existe para ayudar a las personas a detenerse unos minutos en medio del ruido cotidiano para pensar con mayor claridad, crecer con propósito y tomar mejores decisiones.

No pretende decirle a la gente cómo vivir.

Pretende ofrecer ideas que ayuden a mirar la vida desde otra perspectiva.

Cada reflexión también representa mi propio proceso de aprendizaje.

No escribo porque tenga todas las respuestas.

Escribo porque sigo aprendiendo.

---

# 2. Visión

Construir el espacio de desarrollo personal en español con la mejor experiencia de lectura de Internet.

No queremos ser el blog con más artículos.

Queremos ser el lugar donde las personas disfrutan leer, reflexionar y regresar.

---

# 3. Misión

Crear reflexiones profundas, claras y útiles que inspiren cambios reales en la vida cotidiana.

Cada publicación debe aportar valor incluso si el lector solo dedica cinco minutos.

---

# 4. Nuestra promesa

Cuando alguien visite este sitio debe sentir:

• tranquilidad
• claridad
• esperanza
• cercanía
• autenticidad
• elegancia
• inspiración

Nunca debe sentirse presionado.

Nunca debe sentirse manipulado.

Nunca debe sentirse dentro de una página hecha para venderle algo.

---

# 5. Filosofía

No escribimos para obtener clics.

Escribimos para generar reflexión.

No buscamos tiempo en pantalla.

Buscamos tiempo de calidad.

No buscamos visitas.

Buscamos lectores que quieran volver.

No buscamos viralidad.

Buscamos impacto.

---

# 6. Principios del producto

Antes de agregar cualquier funcionalidad debemos preguntarnos:

¿Hace más fácil reflexionar?

¿Hace más agradable leer?

¿Reduce fricción?

¿Hace sentir calma?

¿Aporta valor real?

Si la respuesta es "no", probablemente no deba implementarse.

---

# 7. Público

No escribimos para todo el mundo.

Escribimos para personas que desean crecer.

Profesionales.

Estudiantes.

Emprendedores.

Padres.

Personas que buscan vivir con mayor propósito.

---

# 8. Personalidad de la marca

La marca debe sentirse:

humana

cercana

profunda

elegante

serena

inteligente

honesta

Nunca:

arrogante

agresiva

dramática

sensacionalista

vacía

---

# 9. Estilo de escritura

Siempre escribir:

en español claro

con frases naturales

sin palabras rebuscadas

sin sonar como IA

sin escribir para impresionar

Escribir como una conversación inteligente.

No como una conferencia.

---

# 10. Principios de UX

La lectura es la prioridad.

Todo lo demás es secundario.

Reducir clics.

Reducir distracciones.

Reducir ruido visual.

Usar mucho espacio en blanco.

Tipografía cómoda.

Excelente contraste.

Animaciones discretas.

No usar modales innecesarios.

No usar popups.

No usar banners invasivos.

---

# 11. Home

La Home no es una lista de artículos.

La Home es una experiencia.

Debe responder rápidamente:

Quién soy.

Qué encontrará el visitante.

Por qué vale la pena quedarse.

Debe incluir:

Hero.

Reflexión destacada.

Reflexiones anteriores.

Sobre mí.

Redes.

Footer.

---

# 12. Sobre mí

No es un currículum.

No es una biografía.

Es una conversación.

Debe transmitir humildad.

Autenticidad.

Y explicar por qué comparto estas reflexiones.

---

# 13. Redes sociales

Las redes existen para seguir la conversación.

No para presumir seguidores.

No deben distraer.

Deben complementar la experiencia.

---

# 14. Ingeniería

La arquitectura debe favorecer:

simplicidad

mantenibilidad

escalabilidad

componentes pequeños

responsabilidad única

tipado fuerte

código limpio

No duplicar lógica.

No escribir código "ingenioso".

Preferir claridad.

---

# 15. Decisiones técnicas

Supabase es la única fuente oficial de contenido.

Cloudflare Pages es el entorno de despliegue.

Astro es el framework principal.

TypeScript obligatorio.

Tailwind para estilos.

---

# 16. Antes de aceptar cualquier cambio

Toda modificación debe responder:

¿Mejora la experiencia del lector?

¿Mejora el mantenimiento?

¿Hace el código más simple?

¿Hace el sitio más rápido?

¿Respeta la identidad de la marca?

Si alguna respuesta es negativa, reconsiderar la implementación.

---

# 17. Definición de éxito

Una funcionalidad solo se considera terminada cuando:

funciona

es intuitiva

es accesible

es rápida

es mantenible

respeta este manifiesto

---

# 18. Principio final

La tecnología nunca será el protagonista.

El protagonista siempre será la reflexión.

Todo lo demás existe únicamente para ayudar a que esa reflexión llegue mejor al lector.

# 19. Regla de oro

Si existe una decisión entre:

- más funciones

o

- una mejor experiencia de lectura

siempre elegiremos la mejor experiencia de lectura.

El éxito de este proyecto no se medirá por el número de páginas, botones o funcionalidades.

Se medirá por:

- el tiempo que las personas permanecen leyendo;
- el porcentaje de reflexiones terminadas;
- el deseo de volver;
- la cantidad de personas que recomiendan el sitio;
- el impacto positivo que cada reflexión genera.

Todo cambio futuro debe contribuir a esos objetivos.
