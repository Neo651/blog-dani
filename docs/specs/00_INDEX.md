# Bienvenido

Esta carpeta contiene toda la documentación oficial del proyecto.

Antes de modificar cualquier archivo del código, todo desarrollador o Inteligencia Artificial debe leer esta documentación.

Estos documentos representan la fuente oficial de verdad del proyecto.

Si existe una contradicción entre el código y esta documentación, debe informarse antes de realizar cambios importantes.

---

# Objetivo

Esta documentación busca garantizar que el proyecto mantenga la misma calidad, filosofía, arquitectura y experiencia independientemente de quién lo desarrolle.

La documentación debe evolucionar junto con el producto.

# Reglas

Nunca modificar arquitectura sin justificarlo.

Nunca romper la experiencia del usuario.

Nunca agregar dependencias innecesarias.

Nunca duplicar lógica.

Siempre reutilizar componentes cuando sea posible.

Siempre respetar PROJECT_MANIFEST.
